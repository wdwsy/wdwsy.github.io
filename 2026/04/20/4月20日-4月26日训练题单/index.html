B.CF1844G

考虑树上的路径和为：$dep_x+dep_y-2 \times dep_{lca(x,y)}$
于是我们考虑用二进制来构造，显然在 mod 2 意义下为 $dep_{i+1} = d_i - dep_i$
接下来构造出 mod 4 意义下的值
$dep_{i+1} = (d_i - dep_i + 2 \times dep_{lca(i+1,i)}) % 4$
接下来再构造更高二次幂模意义下的值。

C.P15587

对于这种输出排序结果的问题。我们尝试先排序两个位置：a，b。
发现这并不困难，我们可以找到 c,d，$val_c \leq val_d$，以 $c$ 为根，直接连接除了 $b$ 之外的所有点。然后连接 $a \to b$

  • 最大独立集里面不存在 $c$，返回出来 $a$ 则 $a$ 大，不然的话 $b$ 更大
  • 如果返回了 $c$，那么 $val_a \leq val_b$，因为在这样的独立集中我们可以推断 $val_a+val_d \leq val_b+val_c$

这样也可以同时判断多个二元组 $(a_1,b_1),(a_2,b_2), \dots$ 的大小，于是我们使用奇偶归并排序就可以解决，注意最后我们要把钦定的 $c,d$ 放回去的时候，需要把两次二分的过程并行，每次判断 $(mid_1, c), (mid_2, d)$ 两个二元组的结果才能满足次数限制。

D.P12402

首先，容易得到 $a_i$ 为奇数的时候无解。接下来我们证明偶数都可以找到构造。

我们考虑递归去构造，每次把 $n$ 的规模减少 $1$，接下来我们分两部分来构造，如何化归成子问题，如何构造出当前位置的解。

基本观察

因为要递归成子问题。于是我们考虑消除 $n$ 个二进制位中的一位。
每次回溯的时候，我们会得到若干个数对 $(x,y)$ 满足 $popcount(x \oplus y)=1$， 而 $x,y$ 这两个数字会变成 $x<<1, x<<1|1, y<<1, y<<1|1$ 四个数字，我们考虑这四个数字组合成两个数对，这些数对仍然满足 $popcount()=1$ ，有两种组合方式。

  • $(x<<1,y<<1)$ 和 $(x<<1|1,y<<1|1)$ 这样的话两个数对的异或保持和原来一致。(1)
  • $(x<<1, x<<1|1)$ 和 $(y<<1,y<<1|1)$ 这样的话就是两个数对的异或的二进制位置都变成当前位。(2)

化归

所以我们呢，考虑消除一位，那么这一位上的 $a_i$，必须通过下面的在回溯的时候用 (2) 来得到。
于是就要把每个这个位置上的 $a_i$ 摊到 其他的位置上。
但是递归的时候每个位置上的 $a_i$ 需要处以 $2$，但是如果处以 $2$，之后为奇数就会导致这样的子问题无解，于是我们优先考虑把这样的位置 $+2$。

  • 但是会不会出现，要加上 2 的位置过多，当前位置的 $a_i$ 不够分呢？当然是会的。于是我们贪心的每次删除 $a_i$ 最大的二进制位，如果这样还是不行，可以用数学知识证明，只有在 $n \leq 6$ 的时候会出现。于是我们在 $n \leq 6$ 的时候暴搜就好。
  • 如果 2 还有剩余怎么办？那么我们就随便找一个位置加上就好，因为保证了 $\sum a_i = 2^{n-1}$，可以发现我们递归之后的子问题仍然满足这个条件。

所以每次选择 $a_i$ 最大的二进制位删除，在 $n \leq 6$ 的时候暴力搜索就好。

回溯

根据基本观察，我们可以记录好每次这个二进制位有多少执行 (1)，多少执行 (2)。这样就可以反推出这里的答案。

E.agc_073d

考虑转化这个问题为括号序列，我们之前了解到，对于树的遍历可以看作向下递归为(，回溯过程为)。
这里我们把 $n$ 中字母映射到 $n$ 大类合法的括号序列中。
如何定义大类呢？我们考虑每次询问的时候会给出两个位置 $a_i,b_i$，我们假设给每个位置重新编号为 $rk_i$，$rk_{a_i}<rk_{b_i}$ 就输出 $b_i$（把 $b_i$ 位置当成 1），否则就输出 $a_i$ ，我们可以发现这样输出之后，我们如果把 0 看做左括号，1 看作右括号，一定是一个合法的括号序列。在这样构造下，我们建立了括号序列和合理输出方案之间的映射。

但是是否存在这样的构造呢？我们可以计算一下，长度为 $2n$ 的所有合法括号序列数量为 $\frac{2n \choose n}{n+1}$，所以，我们用这种方法构造 $n$ 次之后，仍然会有 $\frac{2n \choose n}{n+1}$ 次剩余，还是比较宽松的。

于是我们就想要构造出这样的 $n$ 种互斥大类（互斥就是说任意两个大类所包含的合法括号序列之间没有相同的）。我们发现如果一个 $rk$ 的奇数前缀为另外一个 $rk$ 的后缀，那么这两个括号序列一定互斥。原因不难理解。

于是我们想到可以这样构造 $rk$:
$rk_1 = (1, 2, 3, 4, 5, 6, 7, \dots, 2n)$
$rk_2 = (3, 4, 5, 6, 7, \dots, 2n, 1, 2)$
$rk_i = (2i - 1, 2i, \dots, 2n, 1, 2, 2*i - 2)$

这两任意两个位置 $i,j(i < j)$ 都有这样的前后缀 $[2i-1, 2j-2]$。
但是这样，这些公共前后缀的长度总是是偶数，所以我们可以把最前面的数字放到这个数列中 $2*n$ 的后面，即：

$rk_1 = (2, 3, 4, 5, 6, 7, \dots, 2n, 1)$
$rk_2 = (4, 5, 6, 7, \dots, 2n, 3, 1, 2)$
$rk_i = (2i, \dots, 2n, 2i-1, 1, 2, 2i - 2)$
这样他们的公共前后缀就变成了 $[2i, 2*j-2]$。
于是我们就获得了合法的构造。

G.P12402

题意

有一个长度为 $N$ 的整数序列 $A$。求满足以下条件的 $ (1, 2, …, N) $ 的排列 $P$：

  • 对于所有 $i$（$1 \leq i \leq N$），都有 $P_i \leq A_i$。
    请你计算所有满足条件的排列的逆序数之和。由于答案可能非常大，请输出对 $10^9 + 7$ 取模后的结果。

solution

明显的一道数数题。我们发现直接去 dp 满足条件的排列十分困难，于是我们转化为统计贡献。这里套路的，只考虑两个位置上逆序对的贡献，即钦定两个位置 $i,j(i<j)$，求出满足 $a_i > a_j$ 的方案数。

首先我们考虑怎么求总方案数 $tot$，想到高二学长说过的方法，我们考虑限制从紧到松，也就是先填入 $a_i$ 小的位置，于是我们钦定排序好之后的第 $i$ 个位置的限制为 $b_i$，它原来的位置为 $id_i$，这样我们可以得到式子：
$$tot=\prod\limits_{i=1}^{n}(b_i-i+1)$$

接下来我们考虑列出一对位置 $i,j(i<j)$ 的式子，显然我们这样选实质上改变了原来从左向右选择的顺序，本来直观想法应该是不能做的，但是由于逆序对的特殊性，这个时候，$j$ 位置上的选择方案已经与 $[i+1,j]$ 中的选择方案无关了，无论如何，我们都有：
$i$ 选择方案个数：$b_i - i + 1$。
$j$ 选择方案个数： $b_i - i$。

  • $id_i < id_j, i < j$。
    我们发现这样做会影响 $[i+1,j-1]$ 中间位置的选择，他们可以选择的数字都会少一个，因为被 $j$ 位置选过了。因此我们可以找到这一对位置的贡献：
    $$\frac{(b_i - i + 1)(b_i - i)}{2} \times \frac{tot}{(b_i - i + 1)(b_j - j + 1)}\prod\limits_{i<k<j}\dfrac{b_k-k}{b_k-k+1}$$
    $$=\dfrac{tot \times(b_i-i)}{{2}\times(b_j-j+1)}\prod\limits_{i<k<j}\dfrac{b_k-k}{b_k-k+1}$$
  • $id_i > id_j, i < j$
    这样，如果我们按照上面的那个式子来算，我们会算出正序对的数量，于是我们考虑小小地容斥一下，就能得到贡献：
    $$=tot-\dfrac{tot \times(b_i-i)}{{2}\times(b_j-j+1)}\prod\limits_{i<k<j}\dfrac{b_k-k}{b_k-k+1}$$

发现这个式子能够直接用线段树统计，但是其实可以使用树状数组，用树状数组的时候需要判断如果当前 $b_k-k = 0$，那么前面的贡献都会是 $0$，进行一次清空就好。

H.agc025e

做不出来，看题解感觉很诡异。感觉都是注意到有写非人类。

尝试给出一个人类思考路径，首先考虑答案是什么。我们试图去找答案的上界，我们设每个点 $fa_i \to i$ 的这条无向边被这些查询的路径经过的次数为 $p_i$，那么一个显然的上界是 $\sum min(2, p_i)$ 接下来我们考虑构造出这个上界。

一个直观的想法是，我每次从一个点出发，只经过查询边，可以回到自己，那么这个查询路径上的所有路径的贡献都是 2，这样一定是最优的。
从一个点走出去然后回到这个点，显然就是欧拉路径了，而且欧拉路径恰好就是把若干个环拼起来构成的，恰好满足我们希望每个路径上贡献是 2 的要求。但是我们发现如果我们只保留询问边的话可能不存在欧拉路径，于是我们想到，深度从深到低，如果这个位置上的度数为奇数，我们就增加一条 $fa_i \to i$ 的边，这样就能保证每一条边度数都是偶数。而且我们发现这样之后每条边正向经过和反向经过的次数最多相差一，这也就意味着不会改变答案的值。

I.P7361

题意：给你一个字符串 $s$，多次询问 $s[l,r]$ 中至少出现了两次的子串最长是多少。

显然我忘记 $SAM$ 了，所以建立 $SA$。

首先我们考虑二分这个序列，也就是说我们需要判断字符串长度至少为 $mid$ 的字符串是否出现过两次。套路的，我们需要每个以 $i$ 开始的后缀，它后面的第一个 $lcp(i,j) \geq mid$ 的位置 $j$ 是多少。
我们希望在 $O(logn)$ 的时间复杂度内进行一次 $check$，可以用主席树来维护这个序列，我们让 $rt_k$ 对应的主席树为每个位置 $i$ 后面的第一个 $lcp(i,j) \geq k$ 的位置 $j$ 是多少，我们考虑 $k$ 从大到小构建这个主席树，发现每次的操作其实就是把若干个相邻段进行拼接，我们每次可以用 set 记录每个段中的数字，然后每次合并的时候启发式合并（类似于品酒大会的方式）。
