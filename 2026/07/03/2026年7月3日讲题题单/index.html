A - [GCJ 2012 Finals] Shifting Paths

题目概述

你有一张 $n$ 个点的有向图，特殊的，这个图上每一个点只有两个出边，分别为 $i\rightarrow l_i,i\rightarrow r_i$，当你奇数次经过节点 $i$ 的时候，会走向 $l_i$，偶数次经过 $i$ 的时候会走向 $r_i$。
初始的时候位于 $1$，要求输出走多少次之后到达节点 $n$，特殊的，如果无法到达输出”Infinity”。
$n\leq 40$。

做法

首先我们容易发现，我们如果暴力模拟，根据所在的位置 $u$ 和当前每个位置的走的奇偶性集合 $S$，如果走到了相同的状态 $(u,S)$ 那么就说明进入了一个循环，否则我们在 $n2^{n-1}$ 步之内一定能走到 $n$。

观察到 $n$ 很小，于是我们考虑折半，接下来我们可以把所有点平分成两个集合 $A,B$。
我们考虑对于 $A$，预处理出来以每个状态 $(u,S)$ 进入 $A$ 之后，结束的状态 $(u^\prime, S^\prime)$ 以及到达的 $B$ 中的位置，为了保证复杂度，我们从 $n$ 开始 bfs，并记录每个点到 $n$ 的最近距离，我们把距离最小的一半点放入 $B$，其余放入 $A$，这样在 $B$ 中暴力走，复杂度就正确了，达到 $\mathcal{O}(n 2^{n/2})$。

✅ 证明：
我们发现，如果想要走重复的状态，那么必须要两边 $l_i,r_i$ 都至少走过一次，由于我们 bfs 出了这些点，所以满足 $dis_{l_i} < dis_i$ 或 $dis_{r_i} < dis_i$。所以我们最终总会走 $dis$ 越来越小的位置，最后走到 $n$。

G - Joking (Hard Version)

题目概述

这是一个交互题。
你需要猜测在 $1\sim n$ 之间的一个正整数 $x$，$n$ 是已知的，然后每次你可以询问一个集合 $S$，交互库会返回 $x$ 是否在集合 $S$ 中，但是交互库只保证相邻的两次询问中至少一次返回真实结果。
要求在 $53$ 次以内的询问 $S$ 找到这个数字 $x$，你可以猜测不超过 $2$ 次 $x$。
交互库是自适应的。

$n\leq 10^5$。

做题历程

初步思考

首先我们假设我们相邻两次询问为集合 $S,T$，满足 $S\subset T$，我们考虑这个时候交互库会如何返回。

  • $Yes, No:$ 那么 $x \in (\complement_U T) \cup S$
  • $Yes, Yes:$ 那么 $x \in T$
  • $No, Yes:$ 那么 $x \in U$
  • $No, No:$ 那么 $x \in \complement_U S$

因此，一个聪明的交互库，一定会返回 $No,Yes$ 这样就无法缩小集合的范围。

进一步观察，我们可以发现，只要满足下面条件之一，那么我们就会无法缩小集合范围：

  • $S\cup T = U$
  • $S\cup (\complement_U T) = U$
  • $(\complement_U S)\cup T = U$
  • $(\complement_U S)\cup (\complement_U T) = U$

于是我们可以得到结论，如果我们想要缩小集合的范围，那么询问的两个这两个集合总是不互相包含并且有交集。

显而易见的 easy version 做法

全集为 $U$。
在 easy version 中，只要询问 $S \leq 82$ 即可，于是我们可以每两次询问，都把 $S$ 四等分，假设这两次询问的集合为 $S, T$，设 $S\cap T = D$，即 $(\complement_S D) \approx (\complement_T D) \approx D \approx (\complement_U {S\cup T})$。
那么我们两次询问可以把答案集合降低到原来的 $\frac{3}{4}$，这样刚好满足次数限制。

所以 easy version 就解决了。吗？
不，我们发现我们这个做法在 $n=3$ 的时候不奏效了，于是我们需要特殊考虑这种情况。
首先分析我们第一步，它不可能直接猜测 $x$，因为不如先来问一次。

接下来我们可以发现，$n=3$ 的时候，$U = {1,2,3}$，如果我们把这三个数字当成等价的，那么接下来我问任何一个 $U$ 的非空真子集都是本质相同的。因为如果返回了 $x \not\in S$ 相当于是说 $x \in \complement_U S$。于是我们不妨假设第一步询问 $1$，此时如果交互库返回 $Yes$，我们可以直接查验这个位置，如果没有猜中，那么我们知道交互库上一次说谎了，那么下一次它返回的一定是真实答案，那么再令 $S = {2}$，询问一次就行了。但是如果返回了 $No$，我们无法判断交互库是否说谎了，然后我就不会做了，pxb大神告诉我们可以再询问一次 $S = {1}$，如果现在返回的是 $Yes$ 我们可以继续用上面的套路，而如果是 $No$，那么这两个 $No$ 中必须有一次是真的，那么这个 $x \not =1$ 就是已知的了，我们直接判断 $2,3$ 是不是等于 $x$ 即可。

扩展到 hard version

发现 hard version 的询问次数更少，但是我们根据之前的判断：

  • 如果我们想要缩小集合的范围，那么询问的两个这两个集合总是不互相包含并且有交集。

那么我们每次的操作基本固定了，那为什么可以做到更少的询问次数呢？
其实不难发现，我们有一些信息浪费了，具体的，假设我们的询问为 $S_1, S_2, S_3, S_4 \dots$，我们是把 $(S_1, S_2), (S_3, S_4)\dots$，这样分组的，那么如果 $S_2 \cap S_3 \not = \emptyset$，那么其实我们可以利用这个信息进一步缩小范围。那么我们考虑每个 $S_i, S_{i+1}$ 都能缩小范围，这样就不存在浪费了。

错误想法

接下来考虑给每一个 $S_1, S_2, S_3, S_4 \dots$ 分配合适的大小，经过尝试和计算发现无法找到一个平衡的大小。。。

导出正解

所以我们还是考虑 dp 吧，这样只要找出合理的转移，一定是最有的，而且看到这个数字 $53$，我们可以猜测，只要稍微有点不优，可能就通过不了。

于是我们考虑我们归纳这个问题来 dp。

假设现在已经知道了一段前缀 $j$ 的询问，交互库返回了 $Yes$。
我们接下来用这样的方格来表示：

$i$ 标记的部分，表示上一次的询问。

接下来我们考虑这次询问：

我们假定 $i = a+b+c+d,j= a+b$，这次要询问 $b+c$，因为我们已经知道上一次是问了前缀 $i$ 并且返回 $T$，于是我们转移的时候只需要考虑当前位置的返回：

  1. 返回 $Yes$：不难发现，$x\in a\cup b\cup c$，所以 $f_{i,j}\rightarrow f_{a+b+c,b+c}$
  2. 返回 $No$：不难发现，$x\in a\cup b \cup d$，同时评测机认为 $x\not\in b\cup c$，根据之前的结论，这等价于 $x \in a\cup d$，所以 $f_{i,j}\rightarrow f_{a+b+d,a+d}$

⚠️注意：在转移的时候，可能不利用上一次的询问的信息，也就是说可以 $f_{i,j} \rightarrow f_{i,0}$

还有神秘的最后一步，这样 dp 需要枚举 $b,c$ 所以是 2d2d 的，时间复杂度 $\mathcal{O}(n^4)$，$n\leq 10^5$，看起来完蛋了，但是打表半天之后发现如果每次都选择 $b = j / 2, c = (i-j)/2$ 是很优的，于是直接在 $n \geq 100$ 的时候这样交互，剩下的按照 dp 交互就可以通过了，最多正好 $53$ 次。
