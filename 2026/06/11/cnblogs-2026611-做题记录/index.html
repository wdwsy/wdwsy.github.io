序列妙妙值

很好的母题。
我们首先考虑 $dp$。

显然我们可以设 $dp_{i,j}$ 表示，到了位置 $i$，选了 $j$ 段之后的最小和。

暴力转移就是 $O(n^2 k)$ 的做法了。
设 $v = \lfloor log_2(x) \rfloor$。

但是我们注意到 $v \leq 16$，这非常特殊，不是我们想办法让复杂度带上 $v$。

注意到我们的转移方程是
$$dp_{i,j} = \min(dp_{x,j} + (b_{x+1}\ xor \ b_{x+2} \dots \ xor \ b_j))
$$如果我们记录 $s_i = b_1 \ xor\ b_2 \ xor\ \dots \ xor \ b_i$

那么转移方程可以写成：
$$dp_{i,j} = \min(dp_{x,j} + (s_x \ xor \ s_j))
$$于是所有 $s_x$ 相同的位置 $x$，我们就记录最小值就好了，但是这样还是超时。

于是我们发现现在修改我们的记录数组的代价是 $O(1)$ 的，但是每次查询是 $O(2^v)$ 的，于是考虑平衡这两部分的复杂度。
具体的我们可以把每个 $s_x$ 拆成前 $\frac{v}{2}$ 和后 $\frac{v}{2}$ 位考虑，分别记为 $a, b$，我们按照 $a$ 给这些 $s_x$ 分组，然后预处理出来，如果询问的 $s_i$ 的 $b$ 位当前我们枚举的值的时候，我们就会选择位置 $pos[a][b]$ 作为最优解，于是就平衡到 $O(2^{\frac{v}{2}})$ 了。

【MX-X8-T7】「TAOI-3」2236 A.D.

题意简述：

给你一个数，每个节点上有一个颜色，每个颜色有一个权值，输出每一个子树中的所有路径的权值之和，一个路径的权值为这个路径上存在的所有颜色的权值乘积。

我想到了啥

我会暴力！

如果我们要计算子树 $u$ 中的答案，其实每次计算经过了 $u$ 的所有路径和就够了。

但是没有用，我们计算这些还是 $n^2$ 的。

解法

注意到 $k \leq 15$ ，这可能是我们优化的关键。
想要优化复杂度，直观的想法就是先计算重儿子，看看能不能快一点。
首先我们对重儿子 $son$，可以继承他的信息，具体的，对于一个子树中的每个位置，我们可以用一个二进制数表示这个位置到子树的根经过了哪些颜色，然后对于 $2^k$ 种不同的二进制数，我们记录一下各有多少种。
有一个常用的结论，就是每个点的所有轻子树大小之和是 $O(n \log n)$ 的，所以我们可以对轻子树中的位置进行暴力插入。
现在我们看看这个做法复杂度怎么样。
我们进行重剖的预处理部分是 $O(n)$ 的。

然后计算的时候，我们有 $O(n \log n )$ 次插入和查询，其中插入是 $O(1)$ 的，查询（即更新插入一个数字之后的答案）是 $O(2^k)$ 的。

所以总体的复杂度就是 $O(n \log n \times 2^k)$
发现还是不能接受，于是我们可以想到 序列妙妙值 的做法。

我们考虑平衡查询和插入的复杂度，如果他们两个都是 $O(2^{k/2})$ 的，我们总复杂度 $O(n \log n \times 2^{k/2})$ 就足矣通过这个题目了。
接下来我就来详细说说如何完成这个壮举！

我们先把每个二进制数字的二进制位拆开，前 $k/2$ 位为 $a$，后 $k/2$ 位为 $b$，那么我们按照不同的 $a$，把所有可能的二进制数进行分组，然后每次插入的时候，预处理出来，如果查询的值的 $b$ 为 $x$，那么在集合 $a$ 中的数字产生的贡献是多少，这样就可以了。
虽然思路难在这里，但是实际上写起来我把预处理继承的时候的挂载问题调了 3 h，同机房大佬卡常卡了一下午。祭….

龙门考古

题意简述

给你一个长度为 $n$ 的排列 $b$。

现在询问你有多少种选择方式满足：你现在可以选择一些位置留下，其他位置上的数字消失，然后在满足所有前缀最大值的位置不变的前提下，能填出来的最小的排列恰好为 $b$。

做法

官方题解

首先，考虑用一个充要条件来描述合法的选择方式。

设 $c_i=[位置 i 是前缀最大值]$。

  • 假设前缀最大值集合为 $T = {t_1,t_2,\dots}$，那么需要满足 $t_1 < t_2 < t_3 \dots$
  • 所有不选的 $c_i=0$ 的位置的值，都是随着位置递增的（否则可以交换位置）。
  • 虽有不选的 $c_i=1$ 的位置，记下一个 $c_i=1$ 的位置为 $nxt$，$[1,nxt)$ 之间的最大值为 $b_i$，次大值为 $v$，要求 $(v, b_i)$ 之间的值所对应的位置都要选。（否则可以交换位置）。

$tip:$ 显然在第三种情况中，$nxt$ 之前不存在 $(v, b_i)$ 之间的值。
容易验证，这三条就是充要条件。
于是我们可以进行 $dp$，$dp_i$ 表示做到位置 $i$ 的方案数。
我们考虑用一下两种方式来统计方案数。

  • $c_i=0$ 的位置，我们用 $dp$ 统计。
  • $c_i=1$ 的位置，如果在它要求的 $(v, b_i)$ 之间没有选择任何一个，那么它随便选不选，否则必须选。

于是我们就有了 $dp$ 转移：
$$dp_i=\sum\limits_{j=0}^{i-1}\ dp_j \times 2^{f(i,j)}
$$其中 $f(i,j)$ 表示 $(v,b_x) \subseteq (i,j)$ 的区间 $(v,b_x)$ 的个数。
接下来我们用树状数组优化这个过程，注意到所有的 $(v,b_x)$ 不交，发现我们可以把 $f_0$ 设为 $2^{\sum [c_i=0]}$。

然后如果现在是第一次进入当前区间，即 $j < v < i < b_x$ 那么 $dp_i \leftarrow dp_j / 2$，否则 $dp_i \leftarrow dp_j$。
